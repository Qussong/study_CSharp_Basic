## 네임스페이스 
- 간단메모 :
    - 네임스페이스 선언 방법 (namespace)
    - C# 10부터는 파일 범위 네임스페이스 기능을 지원해서 네임스페이스 선언을 간소화할 수 있다.
    - 네임스페이스 사용 방법 (using)
    - 네임스페이스 별칭 : 복잡한 네임스페이스를 사용하는 경우 별칭을 정의하여 코드를 간결하게 할 수 있음
    - 네임스페이스의 장점 (4가지)
    - 콘솔앱 프로젝트 생성시 기본 네임스페이스는 프로젝트 명을 따라가도록 설정되어있다.
    - using static

## 어셈블리 살펴보기 
- 간단메모 :
    - .NET에서 실행가능한 기본 단위
    - 컴파일도니 코드(CIL), 메타데이터, 리소스 등을 포함하는 파일
    - 일반적으로 *.dll 과 *.exe의 확장자를 가진다.
    - .NET 런타임에 의해 로드된다.
    - 실행 어셈블리, 라이브러리 어셈블리
    - 주요 구성요소 : 코드, 메타데이터, 리소스, 어셈블리 매니페스트
    - 강력한 이름과 버전관리
    - 리플렉션

## 어셈블리 생성 및 컴파일 
- 간단메모 :
    - 속성 -> 빌드 -> 출력 -> 기본 출력 경로(bin)
    - 프로젝트 폴더 -> bin -> debug -> net8.0
    - 파일종류 : deps.json, dll, exe, pdb, runtimeconfig.json
    - ildasm.exe
    - JIT 컴파일

## 어셈블리 dll(라이브러리) 참조 
- 간단메모 :
    - 클래스 라이브러리 생성
    - 속성 -> 출력유형 : 클래스 라이브러리 (빌드시 dll 파일 생성되며 해당 기능을 다른곳에서 가져다 사용할 수 있음)
    - 추가 -> 프로젝트 참조 -> 라이브러리 선택 : 종속성/프로젝트 에 해당 라이브러리 클래스 포함된 것을 확인할 수 있다.
    - 종속성 추가시 빌드시 해당 라이브러리도 함께 빌드된다.
    - bin/net8.0 경로에 라이브러리 클래스의 빌드 파일도 함께 있음을 확인할 수 있다.

## 어셈블리 메타정보 
- 간단메모 :
    - 속성 -> 패키지 -> 일반 -> 패키지ID, 제목, 패키지 버전, 작성자, 회사, 제품명, 설명, 저작권(MIT), 프로젝트 URL, ... (메타정보)
    - 프로젝트 파일편집에서 위에 설정해준 정보를 확인할 수 있다.
